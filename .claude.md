# Meal Prep Planner - Claude Instructions

## Project Overview
See @PLANNING.md for the complete implementation guide and @data/README.md for data structure documentation.

## Workflow Preferences
- **No summaries**: Do not generate summaries of changes at the end of tasks
- **Compact context**: When compacting, preserve file paths, constraint definitions, and recipe structure
- **Validation first**: Always validate data changes against the schema before implementing solver logic

## Code Style
- Use Python type hints consistently
- Follow Pydantic best practices for data validation
- Keep YAML files clean and well-commented
- All measurements are RAW (uncooked) ingredients

## Data Model Rules
- Protein types: `chicken`, `beef`, `pork`, `fish`, `egg`
- Carb types: `rice`, `potato`, `platano_verde`, `platano_maduro`, `yuca`, `arepa`, `corn`, `pasta`, `lentils`
- Meal types: `breakfast`, `lunch`, `dinner`
- Carb strategies: `none`, `fixed`, `optional`
- All ingredient IDs are lowercase with underscores (e.g., `fish_salmon`, `platano_verde`)

## Testing
- Validate YAML files before running solver
- Test constraints independently before integration
- Use deterministic seeds for reproducible plans
